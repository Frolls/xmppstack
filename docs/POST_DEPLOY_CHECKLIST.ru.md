# Чеклист После Деплоя

Используйте этот чеклист сразу после первого успешного развёртывания.

## Состояние Сервисов

- `postgres.service` включён и запущен.
- `ejabberd.service` включён и запущен.
- `coturn.service` включён и запущен.
- `nginx.service` включён и запущен, если `xmpp_stack_enable_nginx: true`.
- `nftables.service` включён и запущен.
- `certbot-renew.timer` включён и активен.

## TLS

- Для основного XMPP-домена есть сертификат.
- Для `upload` есть сертификат, если он обслуживается отдельно.
- `conference` покрыт сертификатом, если включён в `xmpp_stack_certbot_domains`.
- HTTPS на публичной точке входа работает.

## Проверка Сети

- `80/tcp` корректно редиректит на HTTPS, если `xmpp_stack_enable_nginx: true`.
- `443/tcp` отвечает корректно, если `xmpp_stack_enable_nginx: true`.
- `5222/tcp` доступен из внешней сети.
- `5269/tcp` доступен, если включена federation.
- `5443/tcp` доступен из внешней сети, если `xmpp_stack_enable_nginx: false`.
- `3478/tcp,udp` и `5349/tcp` доступны для TURN.
- Диапазон TURN relay UDP не режется firewall'ом провайдера.
- `5443/tcp` не торчит наружу, если `xmpp_stack_enable_nginx: true`.

## XMPP-Функциональность

- Логин работает с десктопного клиента.
- Логин работает с мобильного клиента.
- Один пользователь может подключиться с нескольких устройств.
- Архив сообщений работает между устройствами.
- Установка OMEMO-сессии проходит успешно.
- HTTP upload работает.
- Создание комнаты и вход в неё работают.
- Push-уведомления приходят на мобильный клиент.

## Проверка Звонков

- Прямой звонок работает между двумя клиентами из разных сетей.
- Звонок работает, когда хотя бы одна сторона за жёстким NAT или в мобильной сети.
- TURN реально используется, когда peer-to-peer соединение невозможно.

## Проверка DNS

- Основной домен резолвится корректно.
- `upload` резолвится корректно.
- `conference` резолвится корректно, если используется.
- `_xmpp-client._tcp` резолвится корректно.
- `_xmpp-server._tcp` резолвится корректно, если включена federation.
- `_xmppconnect` TXT корректен, если TXT-discovery включён.

## Эксплуатация

- Бэкапы создаются успешно.
- Каталог upload доступен на запись.
- Данные PostgreSQL переживают рестарт сервисов.
- Сертификаты переживают рестарт сервисов.
- Логи читаемы и не содержат явных bootstrap-ошибок.

## Безопасность

- SSH остаётся доступным после включения firewall.
- Снаружи видны только ожидаемые публичные порты.
- `vault.yml` по-прежнему зашифрован и не утёк в логи или plaintext-артефакты.
